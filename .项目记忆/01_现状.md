## 场景模式现状（最近：沉浸增强 + 视角/时空修复，2026-06-30）
- **进出与信号源**：QQ 私聊发「场景模式 / 双人模式 / 退出场景」进出（`scene-commands.ts`）。沉浸态 = 内存 `getSceneMode`（进程清空）‖ DB `activeSceneId`（持久）；`persona-bridge` 一处算 `immersiveMode` 随 result 回传，发送层据此拆条（详见 02 决策，已删回灌）。
- **沉浸输出**：用【】整段包旁白、与对话空行分段、放开篇幅；双人模式旁白写双方动作/反应、用户输入只当剧情引导（不替用户写台词）；场景语音=剥【】发文字→再发语音（语音只读对白）。
- **视角 POV 锁定**：所有【】旁白强制从敏子（用户）视角写——`buildImmersiveOverlay`★铁律＋机械起句口令＋范本全部「敏子起句」；`persona-text-chat` 节奏指令加 POV 限定。
- **时空默认同处**：场景模式默认两人此刻同处一地、面对面，异地设定一律豁免；只有用户给隔空线索（电话/视频/各自在家/异地想念）才按隔空写。两层实现：①`buildEffectiveLifeScheduleOverlay(..., immersiveMode)` 源头换掉异地行为禁令＋加实时定位豁免；②场景 overlay 顶部「时空铁律一·默认同处（类别兜底）＋铁律二·隔空例外（平级）」；③`affectionInstruction` 加 `immersiveMode` 守卫。（详见 02 决策）
- **`/long<N>`**：触发「不要停」连续推进模式，N 段（1-15，缺省 3）。

## 拍照功能现状（LLM 通用拍照，tsc/vitest 全绿）
- **主路径 = LLM 通用拍照**：人物回复里自然输出 `[[PHOTO|带人=…|在家=…|画面=…]]` 标记 → `photo-intent.ts` 解析剥离 → `message-handler.ts` 异步发图 → `selfie-provider.generatePersonaPhoto`（带人/在家/外面三态，附对应参考图）。
- **兜底 = 规则明确指令**：用户明确说「发自拍 / 拍家里」→ `decideSelfieOpportunity` 必发、破冷却。
- **三段式发送**：文字「预告」→（网页生图一两分钟）→ 实拍 → 「交付语」（`selfie-delivery.ts`，selfie/environment/scene 三类话术池、避免连续重复）。
- **区域库**：`PERSONA_SELFIE_HOME_DIR` 放 bedroom/kitchen/livingroom/study/bathroom/entry/balcony.png，`resolveHomeRegionRef` 按场景关键词+作息选（深夜→卧室）。
- **门控**：冷却（3h / 每日 2 张）+ 作息（深夜等静默时段不主动注入拍照标记）。开关 `PERSONA_SELFIE_ENABLED`（默认关）。生图经本机 `chatgpt-project-prompt-pusher`（.NET `--generate` 推网页版 ChatGPT，独占 Chrome profile `google-ai-persistent-profile`，仅 SFW）。
