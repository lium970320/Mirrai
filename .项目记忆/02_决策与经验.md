# 02 · 决策与经验（Mirrai）

## 决策（改写取代，不留矛盾）
- **拍照触发：LLM 标记为主、规则为兜底**。早期纯正则判断「要不要拍」太死板（近似精确匹配才触发）。改为让 LLM 在合适场景自己输出 `[[PHOTO]]` 标记（自然、通吃任意场景），规则只保留「明确指令必发」兜底。
- **发图三段式：预告 → 实拍 → 交付语**。网页生图要一两分钟；若文字即时说「看清楚了吧」会和晚到的图逻辑倒置。故 prompt 约束人物先只「预告」（等下给你拍），图真发出时系统再补一句「交付语」闭环。
- **深夜静默不主动拍**：睡眠/非急事静默时段不向 LLM 注入拍照标记指令（明确叫醒/明确指令仍可）。
- **环境照按区域选参考图**：单一「客厅」参考图会让深夜也贴客厅；改为区域库按场景关键词+作息选（深夜→卧室、做饭→厨房…）。
- **交付语用话术池 + 避免连续重复**、不调 LLM（零延迟），按 selfie/environment/scene 分组（scene=外景/物，话术不提「家」以免出戏）。
- **场景模式时空：默认两人同处一地、可被剧情覆盖成隔空**。日常模式承认武汉-南京异地；但进场景/双人模式是演线下亲密，却一直被异地设定钉住、写成「电话联系」。根因：异地语义散在【当前生活行程】settingLine、【人物核心画像】硬性事实、【原著/长篇背景】、【连续消息与时间指代】、【深情表达规则】至少 5 处，而场景 overlay 旧版只覆盖「禁止旁白/要简短」、从不解除异地。定案两层：①源头 `buildEffectiveLifeScheduleOverlay(personaData, now, immersiveMode)`——场景模式把 settingLine 的异地行为禁令换成「时空豁免」、并对实时定位（当前状态/行为规则）追加「实时定位豁免」；②场景 overlay 顶部加一对平级铁律——「铁律一·默认同处」（类别兜底：本提示词任何位置出现的异地表述一律改判，并枚举出现位置）＋「铁律二·隔空例外」（用户给任何隔空线索/暗示就改写成电话/视频/异地，判不准按用户线索走）；③表白指令 `affectionInstruction`（persona-turn-planner）加 `immersiveMode` 守卫，场景模式不再命令「异地时怎样想起对方」。全部 gated on immersiveMode，日常路径零改动（life-schedule.test 锁定）。

## 踩坑（整条追加，不精简）
- **重启服务会打断正在进行的拍照**：pusher 是服务进程 spawn 的子进程，重启杀服务会让「等图→发图」的那个进程没了，图发不出。教训：重启前先查有没有 `dotnet ...chatgpt-project-prompt-pusher` 在跑，有就等它拍完。
- **清理浏览器进程别误杀 pusher 的 Playwright 会话**：pusher 复用常驻 daemon 会话（label `chatgpt-pic-work-liumin-openai`，占 `google-ai-persistent-profile`）。误杀会报「Browser not open」（pusher 有自愈会自动重开）。要区分「死孤儿」与「活跃会话」——按 `--profile=google-ai-persistent-profile` 精确匹配，别误伤别的 daemon（如 `bugtest`）。
- **两个 CC 会话并行编辑同一批文件会分叉**：曾出现旧 `generatePersonaSelfie` 一套 vs 新 `generatePersonaPhoto`（LLM 通用）一套**配套并存**，两套签名不兼容、不能混镜像（混了 tsc 崩）。解法：统一到其中一整套切换 + tsc 兜底；共享且签名不变的文件（如 `selfie-commands.ts` 正则）改了两套都受益。
- **放宽触发正则别太宽**：环境正则「外面/窗外」配宽前缀会误吃「看外面下雨没/想看看外面」。强场景词（家里/卧室/客厅…）可宽前缀；泛词（外面/窗外）只接「拍/发」紧邻。
- **明确指令的失败兜底别依赖冷却状态**：不冷却时明确「发自拍」会走 LLM 标记路径、`reason` 被当 `spontaneous`、生成失败就静默没补救。修法：`photoIntent` 分支前先算 `decideSelfieOpportunity`，是明确请求就按 `explicit_request`（失败有补救句）。
- **`pg_ctl stop` 在库已停时返回非 0**，会让重启脚本整体 exit 1，但服务实际起来了——以日志「✅ 3000 起来」为准，别被 exit 1 误导。
- **VoxCPM**：20 步本机易超时、10 步稳定；降噪需 FFmpeg shared DLL（装在 torchcodec 目录）。中文 `.ps1` 要存 UTF-8 BOM 否则 PS5.1 双击乱码。
- **判断「未提交改动」前先 `git fetch` 比对 origin**：同步盘会被 Google Drive 异步改写，CRLF 噪声别当真。
- **分支会堆积——合并后顺手删，删前用 `git cherry` 核查零丢失**：工作流是「一功能/存档点开一分支」，加上两个 CC 会话 + Codex 三方并行各开各的，内容汇进 main 后分支没删就积压（曾攒到 5 个）。① 合并 PR 时用 `gh pr merge <n> --merge --delete-branch` 自动删远端+本地，从源头免堆积；② 事后清理别只信 `git branch --merged`（它按 commit 祖先判定，squash / cherry-pick 进 main 的会被误判为「未合并」）——改用 `git cherry origin/main origin/<b>`：`+` 行=真未并入、`-` 行=patch 已进 main；只要 `cherry +` 全为 0，即可安全 `git push origin --delete <b>` + `git branch -D <b>`，零内容丢失。三点 diff（`main...branch`）的「净差异」看着吓人，其实含老分叉点噪声，别拿它当删除依据。
- **场景沉浸态：别让发送层/生成层各自重算开关，用 result 里的单一真相**：沉浸态两信号源——内存 `getSceneMode`（QQ 命令进出、进程清空）+ DB `activeSceneId`（网页/背景场景持久）。曾为让「只认 getSceneMode 的发送层拆条」与「getSceneMode||sceneOverlay 的生成层」同源，在 persona-bridge 每轮 `sceneOverlay!=null` 就回灌 `setSceneMode(true)`——但**回灌单向只开**：网页取消场景（`activateScene(null)` 只清 DB、不碰内存）后内存残留 true →「已退出场景却仍出【】旁白」。根治：**删回灌**，生成层一处算 `immersiveMode` 随 `result.immersiveMode` 回传，发送层（`sayQqReply`/拍照交付句）改用它（缺省回退 getSceneMode）；重启恢复靠 sceneOverlay 实时读 DB，无需回灌。⚠️ 删回灌**不会主动清已残留的内存 true**，改完**必须重启进程**才即时止血——tsx watch（dev:local 跑 server 用它）改动 server 文件即自动重载、内存随之清空（端口 PID 变=已重载）。诊断先查 DB `SELECT "activeSceneId" FROM personas`（camelCase 带引号）：若为 null 却仍出【】，必是内存残留。
- **原著考据触发词别掺日常高频词**：source-recall 的 `SOURCE_RECALL_TRIGGER` 曾误收「学校/北京/南京/见面」——用户随口「我还在学校呢」「我去北京」就被劫持进原著证据模式 → 资料库必然 miss → 落到 source-grounding 硬编码兜底「这段我不敢乱说，记不准的地方我不能编给你听」直接发出（出戏、答非所问，2026-06-24 实测）。规矩：触发词只放 ①专有名词（老鹰峡/柱子/姚敏/湾子）②回忆词（记得/原著/当年/想起）③具体剧情词（中考/考场/车祸/左臂），**绝不放日常会高频出现的地点/动作词**。另：proactive 主动消息（`ambient-proactive`）易凭空虚构「和用户的共同往事」（如「西北头一回给你剥莲子」），已加 prompt 约束「只写当下真实心情、没把握的旧事宁可不提也不编」。
- **同一约束散落多处时，prompt 里「定向点名」盖不住、要用「类别兜底」**：场景共处修复初版只在 overlay 里点名豁免【当前生活行程】那两句异地话，但「异地」在硬性事实/长篇背景/连续消息指代/深情表达至少 5 处用不同措辞重复，模型照样被带回异地（对抗核验子代理发现，非手测）。教训：①覆盖这类弥散语义要写成「本提示词任何位置出现的 X 表述一律改判」并把出现的标题块枚举出来，别逐句点名；②位置在 system prompt 末尾、占后发优势的「正面命令」（如 turn-planner 的 `affectionInstruction` 拼在 `buildSystemPrompt` 之后）必须单独加 `immersiveMode` 守卫，靠前的 overlay 压不住它；③逃生口/例外要与主规则「同等语气权重」（升成平级铁律、降低触发门槛），否则被一段「铁律/凌驾/默认」轰炸的文本里结尾一句轻描淡写的例外会被模型当边角料忽略。④完整性扫描别只扫 `server/**/*.ts`——人物设定正典在 `docs/wang-pengze-persona-profile.md`，经 `normalizePersonaProfileSections`→`buildPersonaProfilePromptSections` 注入（硬性事实/工作所在地/长篇背景）。
