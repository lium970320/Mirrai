# 人物设定与主动消息

## 回复开头括号

服务器会在网页聊天和微信回复返回前清理开头的动作/旁白括号，例如：

- `（她低头笑了笑）我在。`
- `(轻声) 我在。`
- `【沉默片刻】我在。`

清理后会只发送正文：`我在。`。系统提示词也会要求模型不要把动作旁白放在回复开头。

## 长篇人物设定

人物编辑页里有「长篇人物设定 / 小说资料」字段，适合粘贴从小说中整理出来的目标人物背景、经历、关系、说话习惯和关键剧情。每次对话构造提示词时，后端会把这个字段作为独立的长篇背景注入，最多使用前 32,000 个字符。

建议只保留目标人物相关内容：

- 人物经历、身份、关系网。
- 关键剧情节点和心理变化。
- 口头禅、语气、称呼习惯。
- 绝对不能违背的设定。

如果原文很长，优先整理成「人物专属资料」再粘贴进这个字段；不要把整本小说无筛选地塞进去。

## 对话记忆

用户中心的「AI 设置」里，「上下文消息数」控制每次对话会回看最近多少条聊天记录，当前上限是 100 条。这个设置解决的是近期对话记忆；人物背景和性格应写在人物编辑页的长篇设定里。

社交端和网页端还会额外注入「对话连续性」提示：模型会看到最近对话时间线、上一条用户消息、上一条角色回复，以及是否存在角色刚问过但用户还没有回应的问题。这样主动消息不会把每个时间点当作全新开场，例如 18:00 刚问过吃饭而用户没回，20:30 更应该自然跟进“刚才问你吃饭，你也没回我”，而不是重新问一个相似问题。

QQ/微信文本入口有短时间合并队列：连续发来的短消息会先合并成一段连续话语再交给模型。如果模型思考期间用户又发来新消息，旧回复会被判定为过期，不发送也不写入角色回复记录；下一轮会把上一条角色回复之后累积的未回应用户消息当作一个整体来理解，只生成一次综合回复，避免突然把旧消息逐条补答出来。

## 角色日程

后端内置了王芃泽的默认生活日程和状态指示表，作为第一版真实感约束。每个时间段都有基础状态 ID、状态类型、回复可用性、行为规则和下一步转移提示：

- 工作日：早上去南京研究所，08:00-11:30 上午工作，11:30 午饭，14:00-17:30 下午工作，17:30 后下班回家，晚上在家泡茶、看书、整理资料，23:30 后准备睡觉。
- 周六：家里收拾、看书、买菜或偶尔回所里拿资料，晚上更适合自然聊天。
- 周日：上午在家，下午准备下周研究所材料，晚上收心。
- 睡眠时段：普通 QQ/微信社交消息默认不立即回复；如果用户明显有急事、难受、害怕、明确叫醒或需要陪伴，则允许回复。

当前基础状态包括：

- `sleeping`：睡眠，普通消息静默，急事或叫醒才回复。
- `morning_waking` / `weekend_morning`：刚醒和早晨收拾，能回但语气低缓。
- `commuting_to_work` / `commuting_home`：通勤，适合短促报平安，不展开长篇。
- `working_morning` / `working_afternoon` / `sunday_work_prep`：工作或准备工作，回复应短，像工作间隙抽空看手机。
- `lunch_break` / `dinner_at_home`：吃饭，适合日常关心吃饭、下课、晚饭等。
- `midday_rest` / `weekend_errands`：休息或外出琐事，可以短聊但带当下生活感。
- `evening_home`：晚间家中，最适合自然聊天和语音。
- `night_reading`：夜里看书或收心，语气更安静、更近，但不要每次催睡。
- `pre_sleep`：准备睡觉，短句、低声、准备收尾。

这个日程会注入人物系统提示词，所以普通回复、图片/表情包回复、固定时间主动消息、随机日常存在感消息都会知道当前大致状态。它不会改数据库结构；以后如果需要在网页里编辑一周七天的自定义日程，可以再把这套默认日程升级为 `personaData` 配置。

睡眠时段还有一个短期运行状态：如果用户说“醒醒”“你醒了吗”“别睡”或明显有急事，后端会在 `personaData.runtimeLifeState` 里写入 `drowsy_awake`，默认持续 20 分钟。这个状态下后续普通消息会继续得到回复，但提示词会要求角色保持半睡半醒、迷迷糊糊、低声短句的状态；20 分钟内没有继续聊天则自动回到日程里的睡眠状态。

## 主动消息

人物编辑页里可以开启「主动消息」，添加多个 `HH:mm` 时间点。运行时调度器每分钟检查一次：

- 只有人物状态为 ready 时才会发送。
- 优先发送到该人物已绑定的 QQ 联系人；如果没有 QQ 绑定，才回退到微信绑定联系人。
- 有 QQ 绑定时要求 QQ OneBot/NapCat 已连接；没有 QQ 绑定时要求微信机器人已登录。
- 每个时间点每天只发送一次。
- 内容由模型随机生成，但会使用人物系统提示词和长篇设定，避免脱离人物背景。
- 生成前会回看最近对话时间线，优先延续未回应的问题、刚发生的聊天状态和当前日程，不应重复刚问过的话或和前文行程冲突。

主动消息配置保存在 `personaData.proactiveMessages`：

```json
{
  "enabled": true,
  "times": ["09:00", "22:30"],
  "stylePrompt": "像平时自然想起我一样发一句，不要解释自己是定时消息。",
  "lastSent": {
    "09:00": "2026-05-08"
  }
}
```
