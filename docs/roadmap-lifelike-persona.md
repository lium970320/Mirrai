# 让人物真正像真人 —— 下一步路线图

> 状态：内在状态层（持久化心情 + 自己的一天 + 关心回访）已随 PR #7 合并。
> 本文件列出「要真正像个活人，还缺什么」，按优先级排，供你拍板下一步做哪个。
> 每项标了：现状 / 缺口、怎么做（大致）、价值、工作量。

---

## 已经有的（别重复造）

作息与清醒度（life-schedule、drowsy_awake）、意图与风险规划（turn-planner）、隐藏思考层（reflection）、长期记忆全套（召回/治理/沉淀/每日整理/置顶事实）、亲密度等级、主动消息（定时 + 氛围）、语音/表情策略、拟人发送节奏（多条+打字延迟），以及本次新增的：**持续内在情绪 + 人物自己的一天 + 关心回访**。

## P0 — 人物配置化（去单一人物硬编码）★ 地基

- **现状/缺口**：作息表、常驻地、异地伴侣、原著专有词、性别/代词目前**写死在代码里、只对王芃泽成立**（`life-schedule.ts` 三张作息表、`db.ts` 的 `COMMON_SOURCE_TERMS`、`memory-recall.ts` 代词覆盖、`persona-utils`/`turn-planner` 里"符合王芃泽""敏子在武汉"等文案）。换第二个分身或非男性用户就全错。
- **怎么做**：在 persona 资料里加一块 `lifeConfig`（作息模板 + 常驻地 + 关系对象 + 代词 + 专有词表 + 深情语气基调），由素材分析阶段填充；回复链路从配置读取而非常量；给现有王芃泽生成一份配置做迁移。
- **价值**：兑现产品核心「上传任意素材重建任意分身」。不做这一步，后面所有"像真人"只对一个角色成立。
- **工作量**：大。**需先定配置 schema 的方向（要你拍板），适合作为单独一轮的计划。**

## P1 — 主动消息「有由头」而非定时寒暄

- **现状/缺口**：主动消息按时段+概率随机触发、内容靠 LLM 现编；已能问起到期回访。但真人发消息是因为"今天发生了具体的事"或"想起你具体的事"。
- **怎么做**：把「自己的一天」的 dayContext 事件 + 用户的近况/open_loop 喂进主动消息生成，让它"今天所里…顺口想起你说的…"；偶尔主动分享自己当天的事。
- **价值**：主动消息从"机械问候"变成"有来由的惦记"。**直接接在 PR #7 上，见效快、风险低。**
- **工作量**：中。

## P1 — 关系的张力与修复（情绪的关系维度）

- **现状/缺口**：有亲密度分数、关系事件记忆；内心状态有 valence。但没有显式的"关系温度 / 没修复的别扭"——吵完该冷一阵、和好该回暖，且要跨多轮持续。
- **怎么做**：在内心状态旁加一个轻量 `relationshipTone`（亲近 / 有点别扭 / 冷战中 + 强度衰减），由冲突 / 修复 / 表白事件驱动，影响语气与主动性；复用本次的衰减机制。
- **价值**：关系有起伏和记性，不再每轮重置成"默认恩爱"。
- **工作量**：中。

## P2 — 记忆的显著性与遗忘曲线

- **现状/缺口**：有去重/治理/召回打分，已对到期 open_loop 加权。但记忆不会"自然淡忘"，低价值旧记忆和高价值同权堆积。
- **怎么做**：召回打分引入「时间衰减 × 重要度」的显著性；很久没碰的低重要度记忆降权或自动归档。
- **价值**：记得该记得的、淡忘琐碎的，召回更像人脑。
- **工作量**：中小。

## P2 — 真实的回应节奏与「在忙」

- **现状/缺口**：多条消息有打字延迟，睡眠时段会压制即时回复。但白天工作时段也该"隔一会才回 / 晚点看到"，而不是永远秒回。
- **怎么做**：按作息 availability 给回复加真实延迟与"刚看到"的措辞（非睡眠也能"在忙，晚点回你"）。
- **价值**：可用性像真人，不像随叫随到的机器人。
- **工作量**：小。

## P2 — 主动的多模态（偶尔语音 / 图）

- **现状/缺口**：语音 / 表情目前只被动按策略触发。真人会偶尔主动发条语音、发张"我在干嘛"。
- **怎么做**：主动消息链路按概率 + 情境偶尔走语音 / 表情。
- **价值**：陪伴感更立体。
- **工作量**：小中。

## 我的建议

- 想要「立刻能感到更像人」、且顺着这次成果往下走 → 先做 **P1 主动消息有由头 + P1 关系张力**（都直接复用内在状态层，见效快）。
- 想从根上解决「只对一个角色成立」、为多分身和产品卖点铺路 → 优先 **P0 人物配置化**，但要先单独定配置 schema 方向。

> 你定了哪个，我再进入计划模式把它展开成可执行的实现计划（像内在状态层那样：列文件、改动点、复用项、验证）。
